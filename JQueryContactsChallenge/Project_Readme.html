1. Database Installation

Installation can be done using 3 approaches:
  • Import Data-tier Application File
  • Execute sqlcmd script
  • Manual creation

1.1 Import Data-tier Application File

Using SQL Server Management Studio, you can import that database using Deployment\Db\JQueryContactsChallengeDb.dacpac. See https://technet.microsoft.com/en-us/library/jj554810(v=sql.110).aspx for more details.

1.2 Import Data-tier Application File

Using the sqlcmd utility or by using SQL Server Management Studio, you can execute Deployment\Db\JQueryContactsChallengeDb.dacpac to create the database. See https://msdn.microsoft.com/en-us/library/ms162773.aspx for more details on the sqlcmd utility or https://msdn.microsoft.com/en-us/library/ms174187.aspx for more details on SQLCMD mode in SQL Server Management Studio.

1.3 Manually Create Database

  1. Create database — For this project, the database was named JQueryContactsChallengeDb, but you can name it whatever you like.
  2. Create data table — Execute the script located at JQueryContactsChallengeDb\dbo\Tables\Contacts.sql.
  3. Populate data table — Execute the script located at JQueryContactsChallengeDb\Script.PostDeployment1.sql.

2. Install Website

Installation can be done using 2 approaches:
  • Install web deployment package
  • Deploy web app using Visual Studio

2.1 Import Data-tier Application File

Using IIS Manager, you can import the web application from the file located at Deployment\Web\JQueryContactsChallenge.zip. See http://www.iis.net/learn/publish/using-web-deploy/import-a-package-through-iis-manager for more details.

2.2 Deploy web app using Visual Studio

You can open the project in Visual Studio, right-click on the web application project, and deploy the web application

3. Configure Website

Note: Sometimes, when a web app is installed, the web.config file does not get copied along with the deployment package. You may need to copy the web.config file from the source code, if that is the case.

Set connection string — The only other thing you should have to do, is to modify the connection string, which is located in the web.config file at: /config/connectionStrings/add[@name="JQueryContactsChallengeEntities"].

  • Verify SQL server instance — Look for the text data source=LENNYPC\SQLEXPRESS; and set the text between the '=' and ';' to the actual name of the instance when you installed the database.
  • Verify database name — If you created the database as another name, look for the text initial catalog=JQueryContactsChallenge; and set the text between the '=' and ';' to the actual name of the database you created.
